# <a name="microsoft-project-service-description"></a>Descrição do serviço do Microsoft Project

## <a name="overview"></a>Visão geral

Microsoft Project oferece os seguintes aplicativos para ajudar a atender às necessidades da sua organização para gerenciamento de projetos e trabalho:

- Project para a Web
- Project Online 
- cliente de desktop Project Online

Este artigo ajudará você a entender quais aplicativos são fornecidos em cada Project assinatura e quais recursos cada aplicativo oferece.

### <a name="project-for-the-web"></a>Project para a Web

Project para a web é a oferta mais recente da Microsoft para trabalho baseado em nuvem e gerenciamento de projetos. Project para a web fornece recursos simples e poderosos de gerenciamento de trabalho para atender à maioria das necessidades e funções. Project gestores e membros da equipe podem usar Project para a web planejar e gerenciar trabalhos de qualquer tamanho.

Project para a web é construído na Microsoft Power Platform. A Plataforma de Energia é composta por PowerApps, Power Automate, Power BI e o Microsoft Dataverse. Project para os dados da Web é armazenada no Microsoft Dataverse. 

Project para a web está disponível através de três assinaturas: Project Plano 1, Project Plano 3 e Project Plano 5. Os recursos listados em [Project para os recursos da Web](#project-for-the-web-features) interagem apenas com dados no Project para a loja de dados da Web.

### <a name="project-online"></a>Project Online
 
Project Online é uma solução on-line flexível para Project Gestão de Portfólio (PPM) e trabalho cotidiano. Project Online fornece poderosos recursos de gerenciamento de projetos para planejamento, priorização e gerenciamento de projetos e investimentos em portfólio de projetos &mdash; de quase qualquer lugar em quase qualquer dispositivo. O Project Online pode ser usado por administradores, visualizadores e gerentes de portfólio, gerentes de projeto e de recursos, e líderes e membros de equipe.

Project Online é construído na plataforma SharePoint, e armazena dados no SharePoint data store.

Project Online está disponível através de duas assinaturas: Project Plano 3 e Project Plano 5. Os recursos listados [em Project Online recursos](#project-online-features) interagem apenas com dados no Project Online SharePoint armazenamento de dados.

A funcionalidade de membros da equipe para Project Online está disponível através de uma Project Online Essentials ou Project Plano 1 assinatura. Project Online Essentials é uma assinatura *complementa de* membro da equipe para clientes que têm Project Plano 3 ou Project Plano 5 assinaturas. Project Online As assinaturas essenciais e Project Plano 1 dão aos membros da equipe uma interface web para realizar atualizações de tarefas e tempo para projetos gerenciados em Project Online.

Membros da equipe com Project Online Essentials ou assinaturas Project Plano 1 podem executar as seguintes funções Project Online:

- Use uma interface baseada na Web
- Atualizar tarefas, problemas e riscos
- Enviar quadros de horários
- Compartilhe documentos e colabore com Microsoft Teams ou usuários de Skype for Business

Project Online O Essencial não inclui Project para a web. Project Plano 1 inclui tanto Project para a web quanto para Project Online Essentials.

### <a name="project-online-desktop-client"></a>cliente de desktop Project Online

Microsoft Project Online cliente de desktop é um programa de gerenciamento de projetos que possui agendamento automatizado, gerenciamento de recursos de projeto e relatórios incorporados. Project Online cliente de desktop pode ser usado como um aplicativo autônomo ou pode se conectar a Project Online. Project Online cliente de desktop é a versão de assinatura do cliente de desktop Project Professional e está disponível em Project Plano 3 e Project Plano 5 assinaturas.

### <a name="using-project-for-the-web-and-project-online-together"></a>Usando Project para a web e Project Online juntos

Embora Project para a web e Project Online sejam aplicativos separados, você pode usá-los lado a lado. Você pode ver todos os seus projetos de ambas as aplicações em Project Home. Você pode usar a capacidade Project Roadmap para construir roteiros que incluem projetos de ambas as aplicações. E você pode usar Power BI e o Project para o pacote de conteúdo web Power BI para obter insights sobre projetos e recursos em todos os dois aplicativos. <sup>3</sup>

Este diagrama mostra como as aplicações podem trabalhar juntas lado a lado.

![Diagrama mostrando Project na web e Project Online juntos](../media/project-online-applications.png)

Daqui para frente, a Microsoft se concentrará em Project para as inovações da web. No entanto, você pode continuar usando Project Online com confiança, pois continuará recebendo melhorias importantes de desempenho e segurança.

## <a name="service-considerations"></a>Considerações de serviço

> [!NOTE]
> Project para a web está atualmente indisponível para GCC, GCC High e DoD. Estamos trabalhando na entrega de Project para a web para você, mas não podemos compartilhar um prazo firme no momento.

### <a name="licensing-considerations"></a>Considerações sobre licenciamento

- Qualquer interação em um site de Project Online requer pelo menos uma assinatura Project Plano 3 ou Project Plano 5 dentro do inquilino.
- Quando sua última Project Plano 1, Project Plano 3 ou Project Plano 5 assinatura expirar, seu Project para as instâncias da Web não será automaticamente excluído até que você não tenha assinaturas ativas que dependam do Microsoft Dataverse.
- Quando sua última Project Plano 3 ou Project Plano 5 assinatura expirar, suas Project Online instâncias serão excluídas após 120 dias.
- Para Project Online assinaturas de teste, suas instâncias de teste serão excluídas 30 dias após o término do período de teste.
- Para Project para as assinaturas de teste da Web, suas instâncias de teste não serão excluídas até que você não tenha assinaturas ativas que dependam do Microsoft Dataverse.

### <a name="use-of-sharepoint-online"></a>Usar o SharePoint Online

Project Online requer o uso de SharePoint Online, que é provisionada como parte de Project Online. Os direitos à funcionalidade SharePoint Online fornecidos com Project Plano 3 ou Project Plano 5 assinaturas estão limitados ao armazenamento e acesso de dados para Project Online de suporte.

### <a name="project-roadmap-and-power-automate"></a>Project Roteiro e Power Automate

Project O roteiro requer o uso de Power Automate, que é provisionada como parte de sua assinatura de Project. Os direitos de Power Automate funcionalidade estão limitados aos recursos Power Automate exigidos pelo Project Roadmap. Power Automate funcionalidade exigida pelo Roteiro Project aparece como a **Integração de Dados para Project com Flow** plano de serviço na seção **Aplicativos** do centro administrativo Microsoft 365.

### <a name="project-for-the-web-and-microsoft-dataverse"></a>Project para a web e Microsoft Dataverse

Project para a web requer o uso do Microsoft Dataverse para armazenar seus dados. Um banco de dados Microsoft Dataverse é provisionado como parte de sua assinatura Project. Os direitos à funcionalidade do Microsoft Dataverse estão limitados ao armazenamento e acesso de dados para Project de suporte para a web. A funcionalidade do Microsoft Dataverse exigida por Project aparece como o Common **Data Service for Project** service plan na seção **Apps** do centro administrativo Microsoft 365.

Para Project Clientes com cinco (5) ou mais Project para as licenças web, Project para os clientes web podem implantar em ambientes de Produção de Plataformas de Energia e Sandbox.
